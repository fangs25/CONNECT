# CONNECT

CONNECT is a PyTorch-based framework for paired single-cell multi-omics integration, directional cross-modal embedding mapping, and full-feature modality prediction. It is designed for paired RNA+ATAC and RNA+ADT data, with an optional single-modality augmentation workflow for leveraging additional unpaired single-omic cells.

## Overview

Single-cell multi-omics technologies provide paired measurements across molecular layers, but such paired profiles remain expensive and difficult to scale. CONNECT uses limited paired data as a reference bridge to infer missing modalities and translate representations for new cells.

CONNECT provides:

- Directional latent mapping without forcing heterogeneous modalities into a generic shared latent space.
- Full-feature cross-modal prediction without highly variable gene or peak pre-selection.
- Support for RNA+ATAC and RNA+ADT paired datasets.
- Optional single-modality augmentation using a three-stage training workflow.

## 1. System Requirements

### Operating systems

CONNECT is written in pure Python and has been tested on:

- **Linux** (Ubuntu 20.04, 22.04) — primary development and test platform
- **macOS** (12 Monterey, 13 Ventura, 14 Sonoma) — CPU-only or MPS backend
- **Windows** — supported via conda but less extensively tested

### Software dependencies

All required packages are listed in `environment.yml`. The key dependencies are:

| Package   | Version tested | Notes                                          |
|-----------|---------------|------------------------------------------------|
| Python    | 3.10          |                                                 |
| PyTorch   | 2.0.1, 2.5.1  | CUDA 11.7 / 12.1 or CPU-only; see install note |
| numpy     | >=1.24        |                                                 |
| pandas    | >=2.0         |                                                 |
| scipy     | >=1.10        |                                                 |
| anndata   | >=0.8         |                                                 |
| scanpy    | >=1.9         |                                                 |
| tqdm      | >=4.60        |                                                 |

### Hardware requirements

- **GPU**: A CUDA-capable NVIDIA GPU with at least 4 GB VRAM is recommended for training. The demo runs comfortably on 2 GB VRAM.
- **CPU-only**: All workloads run on CPU; training is approximately 5–10× slower.
- **RAM**: 8 GB minimum for the demo dataset; 32+ GB recommended for full-scale datasets.
- **Disk**: ~200 MB for the software and demo data; additional space for downloaded published datasets.

## 2. Installation Guide

**Typical install time:** ~10 minutes on a normal desktop computer (depending on network speed for downloading conda packages).

### Step 1: Clone the repository

```bash
git clone https://github.com/fangs25/CONNECT.git
cd CONNECT
```

### Step 2: Create the conda environment

```bash
conda env create -f environment.yml
conda activate connect-env
```

This installs Python 3.10 and all core dependencies (numpy, pandas, scipy, anndata, scanpy, tqdm).

### Step 3: Install PyTorch

Install PyTorch separately according to your operating system, CUDA version, and GPU driver. Choose the command generated by the [official PyTorch installation selector](https://pytorch.org/get-started/locally/):

```bash
# Example: CUDA 11.7
pip install torch==2.0.1 --index-url https://download.pytorch.org/whl/cu117

# Example: CUDA 12.1
pip install torch==2.5.1 --index-url https://download.pytorch.org/whl/cu121

# Example: CPU-only
pip install torch==2.5.1 --index-url https://download.pytorch.org/whl/cpu
```

### Step 4: Verify the installation

```bash
python -c "from connect import build_model, build_paired_loader, get_device; print('CONNECT installed successfully')"
```

## 3. Demo

The repository includes a real RNA+ADT paired dataset (RPE008 cell line) in
`data/RPE008/`.  Because several `.h5ad` files exceed GitHub's 20 MB limit,
they are stored as split parts — the run scripts **automatically reassemble**
them on first use (no manual steps required).

**Expected run time:**
- ~2–5 minutes on a desktop with a modern NVIDIA GPU (e.g., RTX 3060 or better)
- ~10–20 minutes on a modern CPU (e.g., Apple M1/M2 or Intel i7)

### Run CONNECT on the RPE008 demo

```bash
# Standard paired-data training on RNA+ADT data
python run_connect.py --data-dir ./data/RPE008 --save-dir ./demo_output --device 0
```

If no GPU is available, omit `--device` (defaults to CPU) or use `--device cpu`.

The first run will print a message about reassembling `.h5ad` files from split
parts — this is normal and only happens once.

### Expected output

After training, the output directory (`demo_output/`) will contain:

- `info.log` — Training log with per-epoch loss metrics
- `model_state.pt` — Saved model weights
- `model_outputs.pt` — Inference output tensors (embeddings, mappings, predictions)

**Example training log output:**

```
Reassembling rna_train.h5ad from 21 split part(s) ...
Reassembled rna_train.h5ad (410,779,016 bytes).
Train Epoch: 1 [0/200 (0%)] Loss: 8.123456
    epoch         : 1
    loss          : 6.1234
    ...
Train Epoch: 80 [0/200 (0%)] Loss: 1.234567
    epoch         : 80
    loss          : 1.8456
    ...
```

The loss should decrease over epochs.  Exact numbers vary due to random
initialisation, but a consistent downward trend in `loss` across epochs
indicates correct operation.

## 4. Instructions for Use

### Running on your own data

#### Data format

CONNECT expects paired single-cell data as AnnData (`.h5ad`) files organised by
modality:

```text
/path/to/your/dataset/
  rna_train.h5ad    # RNA training data
  adt_train.h5ad    # ADT training data (or atac_train.h5ad for ATAC)
  rna_test.h5ad     # RNA test data
  adt_test.h5ad     # ADT test data (or atac_test.h5ad for ATAC)
```

- Cells in rows, features in columns.
- Training and test files must have the same cells aligned in order.
- Feature spaces may differ between train and test (the model projects to the
  training feature space).

#### Preprocessing recommendations

| Modality | Setting                              | Description                                |
|----------|--------------------------------------|--------------------------------------------|
| RNA      | `preprocess=False`                   | Raw counts — model handles normalisation   |
| ADT      | `preprocess=True`                    | CLR normalisation                          |
| ATAC     | `preprocess="Andrew"`, `ceiling=True`| TF-IDF with count ceiling                  |

#### Standard paired-data training

```bash
python run_connect.py --data-dir /path/to/your/dataset --save-dir ./results --device 0
```

Key arguments:

| Argument      | Default              | Description                         |
|---------------|----------------------|-------------------------------------|
| `--data-dir`  | *(required)*         | Path to paired `.h5ad` files        |
| `--save-dir`  | `./connect_result`   | Output directory for logs and models |
| `--device`    | `0`                  | GPU index; set to `-1` or omit for CPU |
| `--seed`      | `42`                 | Random seed for reproducibility     |

#### Single-modality augmentation

If you have additional unpaired single-omic cells (e.g., RNA-only data), use the
augmentation workflow:

```bash
python run_unimodal_augment.py
```

Edit the paths at the top of `run_unimodal_augment.py` to point to your data
before running. The three-stage workflow includes:

1. **Stage 1**: Standard paired-data training (80 epochs)
2. **Stage 2**: Augmentation with additional single-modality cells (10 epochs)
3. **Stage 3**: Latent-space alignment fine-tuning (10 epochs)

#### Accessing results programmatically

```python
import scanpy as sc
from connect import (
    build_model, build_paired_loader, get_device,
    init_logger, make_modality, predict, set_seed,
    train_model,
)

# Load your data
rna_train = sc.read_h5ad("rna_train.h5ad")
adt_train = sc.read_h5ad("adt_train.h5ad")

# Preprocess
train_rna = make_modality(rna_train, "RNA", preprocess=False)
train_adt = make_modality(adt_train, "ADT", preprocess=True)

# Build loader and model
loader, valid_loader = build_paired_loader(
    train_rna, train_adt, batch_size=128, validation_split=0.1,
    num_workers=2, training=True,
)
model = build_model(loader, train_rna, train_adt, device="cuda:0")

# Train
model = train_model(model, loader, valid_loader, device="cuda:0", epochs=80)

# Predict on test data
test_rna = make_modality(sc.read_h5ad("rna_test.h5ad"), "RNA", preprocess=False)
test_adt = make_modality(sc.read_h5ad("adt_test.h5ad"), "ADT", preprocess=True)

outputs = predict(model, test_rna, test_adt, device="cuda:0", batch_size=128)

# outputs is a dict with keys:
#   mod1_latent, mod2_latent,
#   mod1_mapping_from_mod2, mod2_mapping_from_mod1,
#   mod1_predicted_from_mod2, mod2_predicted_from_mod1
```
